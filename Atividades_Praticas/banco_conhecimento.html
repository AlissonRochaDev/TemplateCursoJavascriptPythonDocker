O que é HTTP e HTTPS:

HTTP é o protocolo básico para transferir informações na internet, enquanto HTTPS é a versão segura do HTTP que utiliza criptografia SSL/TLS para proteger os dados transmitidos entre o navegador e o site. A principal diferença é a segurança: o HTTPS garante que os dados estejam protegidos contra interceptações, tornando-se essencial para transações sensíveis e para proteger informações pessoais, e é indicado por um ícone de cadeado na barra de URL do navegador.

O que é IP:

IP (Internet Protocol) é um conjunto de regras que permite que computadores e outros dispositivos se comuniquem na internet, atribuindo a cada um um endereço numérico único, semelhante a um "CPF digital". Esse endereço IP, composto por uma série de números separados por pontos (como 192.168.0.1), é fundamental para que os dados sejam enviados e recebidos corretamente, identificando o dispositivo na rede e direcionando-o para o destino correto

O que são Portas:

No contexto de infraestrutura de TI, Portas são pontos de conexão virtuais, identificados por números de 0 a 65535, que permitem a comunicação entre aplicações e serviços dentro de um computador ou rede. Elas funcionam como canais específicos que direcionam o tráfego de dados para o serviço correto, como e-mails (porta 25) ou navegação na web (porta 80), garantindo que a informação chegue ao destino adequado dentro de um sistema.

O que é Docker:

Docker é uma plataforma open-source para criar, executar e gerenciar aplicações através de contêineres. Ele empacota o código de uma aplicação com todas as suas dependências (bibliotecas, ferramentas de sistema, etc.) em um contêiner padronizado. Isso garante que a aplicação funcione de forma consistente e confiável em qualquer ambiente, seja num laptop, numa máquina virtual ou na nuvem. O contêiner é uma unidade executável leve, que compartilha o kernel do sistema operativo do anfitrião, tornando-o mais eficiente que uma máquina virtual tradicional.

O que é Container e Imagem Docker:

Uma imagem Docker é um modelo executável e imutável que contém todas as dependências, bibliotecas e configurações para uma aplicação, funcionando como um snapshot da aplicação. Um contêiner Docker é a instância em tempo de execução de uma imagem, sendo um pacote de software autossuficiente, leve e isolado, que executa a aplicação com base nas instruções da imagem
